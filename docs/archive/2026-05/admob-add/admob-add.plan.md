# Plan: admob-add

## 개요
기출문제 탭(`exam.tsx`)에 AdMob 보상형 광고(Rewarded Ad)를 통합하여
처음 도전하는 기출문제와 재도전 시 광고를 시청해야 입장 가능하도록 한다.

## 배경 / 목적
- 앱 수익화를 위해 보상형 광고 도입
- 사용자가 기출문제를 처음 풀거나 완료 후 다시 풀 때 광고 시청 의무화
- 학습 진행 중(이어서 풀기, 틀린 문제 풀기)에는 광고 없이 자연스럽게 연속 학습

## 광고 노출 조건 (핵심 로직)

| 상황 | 광고 노출 여부 | 설명 |
|------|:---:|---|
| 처음 도전 (`seenCount === 0 && !quizCanResume`) | ✅ | 한 번도 시작한 적 없는 회차 |
| 이미 풀고 있는 중 (`quizCanResume === true`) | ❌ | 진행 중이므로 광고 없이 이어서 |
| 이어서 풀기 (`mode === 'resume-progress'`) | ❌ | 미완료 학습 재개 |
| 틀린 문제 다시 풀기 (`mode === 'incorrect'`) | ❌ | 오답 복습 |
| 전체 다시 풀기 (`mode === 'all'`) | ✅ | 완료 후 재도전 시 광고 필요 |

## 기술 스택 / 라이브러리

- **AdMob SDK**: `react-native-google-mobile-ads` (Expo 호환)
- 기존 스택: Expo (React Native), TypeScript, Zustand, AsyncStorage

## 주요 변경 범위

### 1. 패키지 추가
- `react-native-google-mobile-ads` 설치
- `app.json` AdMob App ID 설정 (Android / iOS)

### 2. 신규 파일
- `features/ads/services/admobService.ts` — 광고 로드·표시 로직
- `features/ads/hooks/useRewardedAd.ts` — 보상형 광고 커스텀 훅

### 3. 수정 파일
- `app/(tabs)/exam.tsx`
  - `handleExamPress`: 처음 도전 분기에서 광고 표시 후 라우팅
  - `navigateWithMode`: `mode === 'all'` 분기에서 광고 표시 후 라우팅
  - 광고 로딩 중 버튼 비활성화 / 로딩 인디케이터 처리

### 4. 광고 흐름
```
[카드 클릭]
  ├─ 처음 도전 → 광고 로드 완료? → 광고 시청 → 퀴즈 진입
  │                              └─ 광고 실패 → (선택) 재시도 or 입장 불가 안내
  └─ 모달 표시
       ├─ 이어서 풀기 → 광고 없이 진입
       ├─ 틀린 문제 풀기 → 광고 없이 진입
       └─ 전체 다시 풀기 → 광고 시청 → 진행 초기화 → 퀴즈 진입
```

## 사용자 경험 고려사항
- 광고 로드 실패 시: 사용자에게 안내 메시지 표시 ("광고를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.")
- 광고 시청 취소 시: 퀴즈 진입 불가 (보상 미지급)
- 광고 로딩 중: 버튼에 로딩 스피너 표시, 중복 클릭 방지

## 제외 범위
- 일반 퀴즈 탭(`quiz.tsx`)에는 광고 미적용
- 플래시카드, 북마크 화면 광고 미적용
- 배너 광고 / 전면 광고는 이번 범위 외

## 완료 기준
1. `seenCount === 0 && !quizCanResume` 케이스에서 보상형 광고 시청 후 퀴즈 진입
2. 모달의 "전체 다시 풀기" 클릭 시 보상형 광고 시청 후 진입
3. "이어서 풀기" / "틀린 문제 풀기" 는 광고 없이 즉시 진입
4. 광고 로드 실패 시 에러 메시지 표시
5. 광고 시청 취소 시 퀴즈 진입 불가
