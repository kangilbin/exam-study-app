# Plan: PDF 이론 데이터 전면 확장

## 개요
- **기능명**: theory-data-expansion
- **목표**: PDF 이론파트(v0.91) 31페이지 전체 내용을 학습 앱 데이터로 변환
- **현황**: 현재 이론 20문제 + 암기 48문제 = 총 68문제 (PDF 내용의 약 15% 수준)
- **목표**: PDF 10개 챕터 + 정리표 전체를 문제화하여 **230문제** 확보 (기존 카테고리 통합 전략)

## 문제 정의
PDF "정보처리기사 실기 대비 이론문제 v0.91"에 10개 챕터(요구사항~프로그래밍 언어) + 정리표(디자인패턴, SQL/JOIN, 네트워크, 신기술용어, 다이어그램 등) 내용이 있으나, 현재 앱에는 극히 일부만 반영됨.

### 구현 전략 변경 (v2 - 2026-04-01)
> 원래 Plan에서는 9개 신규 카테고리를 별도 파일로 생성할 계획이었으나,
> 앱 UI의 카테고리 카드가 과도하게 많아지는 것을 방지하고 코드 변경을 최소화하기 위해
> **기존 4개 theory + 4개 memorize 카테고리에 subcategory를 확장하여 통합**하는 전략으로 변경.
>
> - Ch3(통합구현), Ch4(서버프로그램), Ch5(인터페이스), Ch6(화면설계), Ch7(테스트), Ch12(패키징) → theory-se의 subcategory로 통합

> - 보안/신기술 암기 → memorize-network, memorize-os에 통합
>
> 이 방식의 장점: 앱 코드 변경 불필요(types.ts, questionService.ts 수정 없음), 기존 UI 구조 유지

## 문제 작성 규칙
1. PDF 원문을 최대한 충실히 반영
2. `(기출)` 표시된 내용은 반드시 포함
3. 문제 형식: 기존 JSON 구조 유지 (id, categoryId, subcategory, type, difficulty, question, answer, explanation, choices, source, tags)
4. `source: "pdf"`, `sourceFile: "(유료)_정보처리기사_실기_이론파트_v0.91.pdf"` 고정
5. 4지선다 choices 포함 (theory 타입)
6. 암기 카드는 choices 없이 question/answer만 (memorize 타입)
