## 7. 교훈 (Lessons Learned)

### 7.1 잘된 점 (What Went Well)

✅ **설계 기반 병렬 실행의 효율성**
- 4개 에이전트 병렬 실행으로 총 44분 내 완료 (순차 실행 대비 4배 이상 가속)
- Phase별 분할(241→93→86→103)로 작업 복잡도 관리

✅ **명확한 Design 문서의 가이드 역할**
- Definition 3등급 분류(A/B/C)가 실행 단계에서 큰 혼동 없이 적용됨
- tip 5가지 유형 템플릿이 일관성 있는 출제 경향 정보 생성 가능

✅ **기존 데이터 무결성 보존**
- 523장 전체 id, categoryId, tags, relatedTerms 100% 보존
- 기존 82개 커스텀 두문자어 100% 유지로 학습 연속성 보장

✅ **검증 프로세스의 조기 오류 감지**
- gap-detector 분석으로 SE 15장의 tip 길이 미달을 정량적으로 식별
- 미달 비율(2.9%)이 Low 영향도로 판정되어 Iterate 불필요 결론

### 7.2 개선 영역 (Areas for Improvement)

⚠️ **Tip 길이 기준의 사전 정의 필요**
- Design에서 "50~200자"로 명시했으나 SE 일부 카드에서 실행 시점에 조정됨
- **개선**: Design 단계에서 카테고리별 tip 길이 기준 상세화 (예: 패턴류는 40자, 프로토콜은 60자)

⚠️ **병렬 실행 시 품질 검증 강화**
- 4개 에이전트 동시 실행으로 개별 카드 검토 깊이가 다소 불균형
- **개선**: Phase 2~4 시작 전 Phase 1 샘플링 검증(10%) → 이후 Phase에 피드백 반영

⚠️ **기출문제 정합성 검증 부재**
- tip 내용이 실제 기출 경향과 정합하는지 사후 샘플링 검증 미실시
- **개선**: 정보처리기사 기출 DB와 매핑하여 tip 신뢰도 검증 프로세스 추가
