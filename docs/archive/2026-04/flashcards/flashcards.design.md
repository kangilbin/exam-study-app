# Design: flashcards 암기카드 콘텐츠 보강

> Plan 참조: `docs/01-plan/features/flashcards.plan.md`

## 1. 데이터 스키마

### 카드 JSON 구조 (보강 후)

```jsonc
{
  "id": "card-memorize-{과목}_{번호}",       // 변경 금지
  "categoryId": "memorize-{과목}",            // 변경 금지
  "subcategory": "서브카테고리",               // 변경 금지
  "term": "용어/문제",                         // 변경 금지
  "definition": "보강된 설명",                 // ★ 보강 대상
  "mnemonic": "암기 TIP",                     // ★ 추가 (이미 다른 값이 있으면 유지)
  "tip": "시험 출제 경향 + 암기 요령",         // ★ 추가 대상
  "source": "memorize",                       // 변경 금지
  "sourceQuestionId": "memorize-{과목}_{번호}", // 변경 금지
  "tags": ["태그"],                            // 변경 금지
  "relatedTerms": ["관련카드id"]               // 변경 금지 (있는 경우만)
}
```

### 필드별 규칙

| 필드 | 작업 | 규칙 |
|------|------|------|
| `definition` | 보강 | 짧은 설명 → 이해 가능한 수준으로 확장. 이미 충분하면 오탈자만 수정 |
| `mnemonic` | 추가/유지 | 기존에 두문자어 값("도부이결다조" 등)이 있으면 그대로 유지. 없거나 빈 값이면 `"암기 TIP"` 설정 |
| `tip` | 추가 | 전체 카드에 시험 출제 경향 + 오답 패턴 + 암기 요령 작성 |
| 기타 필드 | 변경 금지 | id, categoryId, subcategory, term, source, sourceQuestionId, tags, relatedTerms |

## 2. definition 보강 규칙

### 2.1 보강 등급 분류

| 등급 | 기준 | 작업 |
|------|------|------|
| A (대폭 보강) | definition 50자 미만 | 전면 재작성 — 용어 풀이 + 개념 설명 + 특징/비교 |
| B (부분 보강) | definition 50~150자 | 부족한 부분만 보충 — 예시, 비교, 키워드 추가 |
| C (최소 수정) | definition 150자 이상 | 오탈자 수정, 구조 정리만. 내용 축소 금지 |

### 2.2 definition 작성 포맷

```
{영문 풀네임} ({한글 풀이})

{1~2줄 핵심 개념 설명 — "쉽게 말해..." 수준}

[특징] 또는 [구성요소] 또는 [비교]
• 포인트 1
• 포인트 2
• 포인트 3
```

**규칙:**
- 첫 줄: 영문 약어가 있으면 풀네임 + 한글 뜻
- 본문: 초보자도 이해할 수 있는 쉬운 말로 작성
- 기존에 암기 키워드(두문자어)가 있으면 반드시 유지: `(암기: 용단피의존)` 등
- 줄바꿈은 `\n` 사용 (JSON 문자열 내)
- `•` 불릿 사용 (기존 카드와 동일)
- 너무 길지 않게 — 학습카드이므로 핵심만 (최대 500자 내외)

## 3. tip 작성 규칙

### 3.1 tip 작성 포맷

```
{출제 경향 1줄} + {오답 패턴 또는 함정 1줄} + {암기 요령 1줄}
```

**규칙:**
- 최소 1줄, 최대 3줄 (50~200자)
- 반드시 시험과 연관된 실전 정보 포함
- "~가 오답으로 자주 나옵니다" / "~와 혼동하지 마세요" 형태 권장
- 두문자어가 있으면 tip에서도 강조

### 3.2 tip 유형별 템플릿

| 유형 | 적용 대상 | 템플릿 |
|------|----------|--------|
| 오답 경고형 | 혼동하기 쉬운 개념 | "시험에서 {A}를 {B}라고 바꿔 출제합니다. {A}={핵심}임을 기억하세요!" |
| 순서 암기형 | 단계/순서가 있는 개념 | "{두문자어}로 외우세요! 순서를 바꿔 출제하는 함정에 주의!" |
| 비교 구분형 | 유사 개념이 있는 카드 | "{A}와 {B}의 차이: {A}는 {특징1}, {B}는 {특징2}. 이 차이가 시험 포인트!" |
| 키워드 연결형 | 특정 키워드=특정 개념 | "'{키워드}'가 보이면 무조건 {개념}! 다른 보기에 속지 마세요." |
| 계산 공식형 | 공식이 있는 카드 | "공식: {공식}. 시험에서는 {변형}으로 물어봅니다. 단위에 주의!" |

### 3.3 mnemonic 필드 처리

```
기존 mnemonic 값 있음 ("도부이결다조", "물데네전세표응" 등)
  → 그대로 유지 (두문자어 암기용)

기존 mnemonic 값 없음 또는 빈 문자열
  → "암기 TIP" 으로 설정
```

## 5. 검증 규칙

### JSON 유효성 검증

각 파일 수정 후 반드시 실행:

```bash
python3 -c "import json; json.load(open('{파일}')); print('OK')"
```

### 필드 무결성 검증

수정 후 검증할 항목:

```bash
# 1. 카드 수 변경 없음 확인
python3 -c "import json; print(len(json.load(open('{파일}'))))"

# 2. 모든 카드에 tip 필드 존재 확인
python3 -c "
import json
data = json.load(open('{파일}'))
no_tip = [c['id'] for c in data if not c.get('tip')]
print(f'tip 없는 카드: {len(no_tip)}')
"

# 3. 기존 id 변경 없음 확인 (수정 전/후 id 목록 비교)
python3 -c "
import json
data = json.load(open('{파일}'))
ids = [c['id'] for c in data]
print(f'중복 id: {len(ids) - len(set(ids))}')
"
```

## 6. 주의사항

1. **정확성**: 정보처리기사 실기 기출문제와 교재를 기준으로 정확한 내용만 작성
2. **일관성**: 같은 서브카테고리 내 카드들의 설명 깊이와 형식을 통일
3. **특수문자 이스케이프**: JSON 내 `"` → `\"`, 실제 줄바꿈 → `\n`
4. **기존 두문자어 보존**: "도부이결다조", "물데네전세표응" 등 기존 mnemonic 값 유지
5. **relatedTerms 활용**: 관련 카드가 있으면 definition에서 비교/구분 내용 포함
