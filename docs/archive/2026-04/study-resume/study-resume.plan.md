# Plan: study-resume (학습탭 이어서 학습 기능)

## 개요
- **기능명**: study-resume
- **유형**: UX 개선
- **우선순위**: 높음
- **작성일**: 2026-04-14

## 배경 및 목적

현재 **기출문제탭**에는 학습 기록이 있을 때 "이어서 풀기 / 전체 다시 풀기" 모달이 있어 사용자가 학습 흐름을 이어갈 수 있다. 하지만 **학습탭**에는 진척도(프로그레스 바)만 표시될 뿐, 이어서 학습하는 기능이 없어 매번 처음부터 시작해야 한다.

또한 카드 학습 중 **뒤로가기 버튼**을 누르면 이전 카드가 아닌 화면 자체에서 빠져나가는 문제가 있어, 학습 중 이전에 본 카드를 다시 확인할 수 없다.

### 현재 문제점

1. **학습탭에 이어서 학습 없음**: 카테고리 클릭 시 바로 학습 화면으로 이동하여 항상 처음부터 시작
2. **카드 학습 중 이전 카드 볼 수 없음**: `markKnown`/`markUnknown`이 `currentIndex`를 증가만 시키고, 뒤로 돌아가는 방법 없음
3. **뒤로가기 = 화면 이탈**: 카드 학습 중 뒤로가기(`handleGoBack`) → `fcResetSession()` → `router.back()` 호출로 화면에서 완전히 빠져나감
4. **세션 미저장**: `useFlashcardStore`가 `cardProgress`와 `displayMode`만 영속화하고 세션 상태(어디까지 봤는지)는 저장하지 않음

### 참고: 기출문제탭 기존 동작 (exam.tsx)
- 학습 기록 없으면 → 바로 진입
- 학습 기록 있으면 → 바텀시트 모달 표시
  - 진행 상태 (전체/학습완료/미학습) 또는 점수 표시
  - "이어서 풀기" / "전체 다시 풀기" 버튼
  - 완료 시 "틀린 문제만 다시 풀기" 추가

## 요구사항

### FR-01: 학습탭 카테고리 클릭 시 모달 표시

학습탭(`index.tsx`)에서 카테고리를 클릭할 때, 학습 기록이 있으면 기출문제탭과 유사한 바텀시트 모달을 표시한다.

**암기 카테고리 (memorize-\*):**
| 조건 | 동작 |
|------|------|
| 학습 기록 없음 (cardProgress에 해당 카테고리 카드 없음) | 바로 학습 화면 진입 |
| 학습 기록 있음 | 모달 표시 |

**모달 표시 내용:**
- 카테고리명 + 아이콘
- 진행 상태: 전체 카드 수 / 학습완료(known) / 미학습(unseen+unknown)
- 버튼:
  - **"이어서 학습"** (primary): 마지막으로 본 카드 다음부터 시작
  - **"처음부터"** (outline): 전체 카드를 처음부터 다시
  - **"모르는 카드만"** (outline, unknown > 0일 때): unknown 카드만 필터링하여 학습
- 취소 버튼

**알고리즘/일반 카테고리 (code-\*, sql-\*, theory-\*):**
| 조건 | 동작 |
|------|------|
| 학습 기록 없음 (seenCount === 0) | 바로 학습 화면 진입 |
| 학습 기록 있음 | 모달 표시 |

**모달 표시 내용:**
- 카테고리명 + 아이콘
- 진행 상태: 전체 / 학습완료 / 미학습
- 버튼:
  - **"이어서 학습"** (primary): 안 푼 문제부터 시작 (mode=unseen)
  - **"처음부터"** (outline): 전체 다시 풀기 (mode=all)
  - **"틀린 문제만"** (outline, incorrectCount > 0일 때): 틀린 문제만 (mode=incorrect)
- 취소 버튼

### FR-02: 카드 학습 중 이전 카드 보기

카드 학습 화면에서 이전에 본 카드로 돌아갈 수 있도록 한다.

**네비게이션 방식:**
- 카드 아래 영역에 **"< 이전"** / **"다음 >"** 버튼 추가
- "이전" 버튼: 이전 카드로 이동 (단순 조회, 암기 상태 변경 없음)
- "다음" 버튼: 다음 카드로 이동 (미응답 카드면 현재 위치 유지)
- 스와이프(알아요/모르겠어요)는 기존대로 유지하되, 이전 카드 재방문 시에도 스와이프로 상태 변경 가능

**상태 관리:**
- `useFlashcardStore`에 `goToPrevious()` 액션 추가
- `currentIndex`를 감소시켜 이전 카드 표시
- 이전 카드의 기존 암기 상태(known/unknown) 표시

### FR-03: 뒤로가기 동작 개선

현재 뒤로가기 시 화면에서 완전히 빠져나가는 문제를 수정한다.

**동작 규칙:**
| 상황 | 뒤로가기 동작 |
|------|--------------|
| 첫 번째 카드 (currentIndex === 0) | 학습 종료 확인 모달 표시 |
| 두 번째 이후 카드 | 이전 카드로 이동 |
| 학습 종료 확인 모달에서 "나가기" | 세션 저장 후 화면 이탈 |
| 학습 종료 확인 모달에서 "계속 학습" | 모달 닫기, 학습 계속 |

**종료 확인 모달:**
- "학습을 종료하시겠습니까?"
- "진행 상태는 저장됩니다."
- [나가기] / [계속 학습]

### FR-04: 세션 상태 영속화

학습 세션의 위치 정보를 저장하여, 앱을 종료하고 다시 들어와도 이어서 학습할 수 있도록 한다.

**저장 대상:**
```typescript
// useFlashcardStore partialize 확장
{
  cardProgress: Record<string, CardProgress>;  // 기존
  displayMode: CardDisplayMode;                 // 기존
  lastSession: {                                // 신규
    categoryId: CategoryId;
    cardIds: string[];    // 카드 순서 보존
    currentIndex: number; // 마지막 위치
    savedAt: string;      // 저장 시각 (ISO)
  } | null;
}
```

**복원 로직:**
- "이어서 학습" 선택 시 `lastSession`의 `cardIds` 순서 + `currentIndex` 위치로 복원
- "처음부터" 선택 시 `lastSession` 초기화 후 새 세션 시작
- 카드 이동(`markKnown`/`markUnknown`/`goToPrevious`) 시마다 `lastSession.currentIndex` 자동 갱신

## 기술 설계 방향

### 수정 대상 파일

| 파일 | 변경 내용 |
|------|----------|
| `app/(tabs)/index.tsx` | 카테고리 클릭 시 모달 로직 추가 |
| `store/useFlashcardStore.ts` | `goToPrevious()`, `lastSession` 영속화, `saveSession()` 추가 |
| `app/quiz/[categoryId].tsx` | 뒤로가기 동작 변경, 이전/다음 버튼 UI 추가 |
| `components/FlashCardDeck.tsx` | 이전/다음 버튼 영역 추가 (선택적) |

### 신규 파일 없음
- 기존 컴포넌트와 스토어를 확장하여 구현
- 기출문제탭의 모달 패턴을 학습탭에 재활용

### 화면 흐름

```
학습탭 카테고리 클릭
  ├── 학습 기록 없음 → 바로 진입
  └── 학습 기록 있음 → 모달 표시
        ├── "이어서 학습" → lastSession 복원하여 진입
        ├── "처음부터"    → lastSession 초기화 후 진입
        └── "모르는 카드만" → unknown 필터링 후 진입

카드 학습 중
  ├── 스와이프 → 알아요/모르겠어요 (기존)
  ├── 이전 버튼 → 이전 카드 조회 (신규)
  ├── 뒤로가기 (첫 카드) → 종료 확인 모달 (신규)
  └── 뒤로가기 (그 외) → 이전 카드로 이동 (신규)
```

## 구현 순서

### Phase 1: 스토어 확장
1. `useFlashcardStore`에 `goToPrevious()` 액션 추가
2. `lastSession` 필드 추가 및 `partialize`에 포함
3. `markKnown`/`markUnknown` 시 `lastSession` 자동 갱신

### Phase 2: 카드 학습 화면 개선
1. 이전/다음 네비게이션 버튼 추가
2. 뒤로가기 동작 변경 (이전 카드 or 종료 확인 모달)
3. 종료 확인 모달 구현

### Phase 3: 학습탭 모달
1. `index.tsx`에 모달 상태 및 UI 추가
2. 암기 카테고리용 모달 (이어서/처음부터/모르는 카드만)
3. 일반 카테고리용 모달 (이어서/처음부터/틀린 문제만)
4. 학습 기록 유무에 따른 분기 처리

### Phase 4: 세션 복원 연동
1. "이어서 학습" 시 `lastSession`에서 복원
2. 화면 이탈 시 세션 자동 저장
3. 엣지 케이스 처리 (데이터 변경 시 카드 ID 불일치 등)

## 범위 외 (Out of Scope)
- 일반 문제(theory/algorithm) 세션 영속화 (이미 `useQuizStore`에 persist 존재)
- 기출문제탭 모달 수정 (기존 그대로 유지)
- 카드 순서 변경(셔플) 옵션
- 스와이프 방향으로 이전 카드 이동 (혼동 방지)
