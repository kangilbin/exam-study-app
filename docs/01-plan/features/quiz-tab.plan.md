# Plan: quiz-tab (퀴즈탭 카드형 암기 모드 전환)

## 개요
- **기능명**: quiz-tab
- **유형**: UI/UX 개선 + 데이터 구조 변환
- **우선순위**: 높음
- **작성일**: 2026-04-13

## 배경 및 목적

현재 퀴즈탭(암기탭)은 **문제 풀이 형식**으로 구현되어 있으나, 실제 목적은 **용어/개념 암기**이다.
암기에 최적화된 **카드형 플래시카드** UI로 전환하여 학습 효율을 높인다.

### 현재 문제점
1. **암기 카테고리도 문제 풀이 형식**: memorize-* 카테고리가 문제/정답 구조로 되어있어 암기 효율이 낮음
2. **중복 콘텐츠 다수**: memorize-* 데이터 내 같은 주제가 난이도만 다르게 중복 존재
   - memorize-db: ACID(001, 012), 이상현상(004, 014), 관계대수(005, 017), 키(008, 018), 회복기법(006, 015)
   - memorize-network: 프로토콜 3요소(002, 024), TCP/UDP(004, 016), DoS(007, 018), 라우팅(006, 022)
   - memorize-os: 교착상태 조건(003, 018), 교착상태 해결(004, 013), 프로세스 상태(005, 016)
3. **복합 문제 미세분화**: 하나의 문제에 여러 용어가 묶여 있어 개별 암기 불가
   - 예: "카디널리티와 차수의 차이" → 카디널리티/차수 각각 별도 카드 필요
   - 예: "SQL 분류(DDL, DML, DCL, TCL)" → 각 분류별 별도 카드 필요
4. **학습탭(theory-*) 보기 용어 미활용**: 객관식 보기에 나오는 용어가 암기 카드로 활용 불가

## 요구사항

### FR-01: 카드형 플래시카드 UI
- memorize-* 카테고리 진입 시 카드형 UI 표시
- **앞면**: 용어(Term)
- **뒷면**: 설명(Definition)
- 카드 탭/스와이프로 앞뒤면 전환
- 좌우 스와이프로 이전/다음 카드 이동

### FR-02: 토글 모드 (용어/설명 반전)
- 상단 토글 버튼으로 표시 모드 전환
  - **기본 모드**: 앞면=용어, 뒷면=설명
  - **반전 모드**: 앞면=설명, 뒷면=용어
- 반전 모드에서는 설명을 보고 용어를 맞추는 방식

### FR-03: 알고리즘 카테고리 주관식 유지
- 알고리즘 관련 카테고리는 카드형이 아닌 **주관식 문제 풀이** 유지
- 대상 카테고리: `code-c`, `code-java`, `code-python`, `code-common`, `sql-dml`, `sql-ddl`, `sql-set`
- 기출문제 탭과 동일한 주관식 입력 → 채점 방식 적용

### FR-04: 중복 콘텐츠 제거 및 통합
- 동일 주제의 중복 문제를 하나로 통합 (상세한 버전 유지)
- 통합 대상 (총 12쌍):
  - **memorize-db**: 5쌍 (ACID, 이상현상, 관계대수, 키, 회복기법)
  - **memorize-network**: 4쌍 (프로토콜3요소, TCP/UDP, DoS, 라우팅)
  - **memorize-os**: 3쌍 (교착상태 조건, 교착상태 해결, 프로세스 상태)

### FR-05: 복합 문제 세분화 (카드 분리)
- 하나의 문제에 여러 용어가 포함된 경우 개별 카드로 분리
- 분리 예시:
  | 원본 문제 | 분리 결과 |
  |-----------|-----------|
  | 카디널리티와 차수의 차이 | 카드1: 카디널리티 → 튜플(행)의 수 |
  |  | 카드2: 차수(Degree) → 속성(열)의 수 |
  | SQL 분류(DDL, DML, DCL, TCL) | 카드1: DDL → CREATE, ALTER, DROP, TRUNCATE |
  |  | 카드2: DML → SELECT, INSERT, UPDATE, DELETE |
  |  | 카드3: DCL → GRANT, REVOKE |
  |  | 카드4: TCL → COMMIT, ROLLBACK, SAVEPOINT |
  | 결합도 종류 | 카드1~6: 각 결합도별 개별 카드 |
  | 응집도 종류 | 카드1~7: 각 응집도별 개별 카드 |
  | GoF 패턴 분류 | 생성/구조/행위 각 패턴별 개별 카드 |
  | OSI 7계층 | 각 계층별 개별 카드 |
  | 보안 솔루션 | 각 솔루션별 개별 카드 |

### FR-06: 학습탭(theory-*) 보기 용어 → 카드 변환
- theory-* 카테고리의 객관식 보기에 등장하는 용어를 암기 카드로 추출
- 추출 기준:
  - 보기 중 정답인 용어 + 해설에서 설명이 있는 오답 용어
  - 이미 memorize-* 에 있는 용어는 제외 (중복 방지)
- 예시:
  - theory-db_001: "차수(Degree)" → 이미 memorize-db에 있으므로 제외
  - theory-db_002: "개념 스키마/외부 스키마/내부 스키마" → 각각 카드로 추출

### FR-07: 암기 상태 추적
- 각 카드별 "알아요" / "모르겠어요" 상태 저장
- 카드 슬라이드 완료 시 자동 기록
- "모르겠어요" 카드만 모아서 복습 모드 지원

### FR-08: 학습 진행률 표시
- 카테고리별 암기 진행률 (알아요 카드 수 / 전체 카드 수)
- 퀴즈탭 카테고리 목록에 진행률 표시

## 데이터 구조 설계

### 새로운 카드 데이터 스키마
```typescript
interface FlashCard {
  id: string;                    // "card-{categoryId}_{number}"
  categoryId: string;            // memorize-se, memorize-db 등
  subcategory: string;           // 소분류
  term: string;                  // 용어 (앞면)
  definition: string;            // 설명 (뒷면)
  mnemonic?: string;             // 암기법 (선택)
  relatedTerms?: string[];       // 관련 용어
  source: 'memorize' | 'theory'; // 원본 출처
  sourceQuestionId: string;      // 원본 문제 ID
  tags: string[];
}
```

### 데이터 변환 파이프라인
```
1. memorize-*.json 읽기
2. 중복 제거 (상세 버전 유지)
3. 복합 문제 → 개별 카드 분리
4. theory-*.json 보기 용어 추출
5. 기존 카드와 중복 체크
6. flashcards-*.json 생성
```

### 예상 카드 수
| 카테고리 | 현재 문제 수 | 중복 제거 | 세분화 후 | theory 추출 | 예상 총 카드 |
|----------|:-----------:|:---------:|:---------:|:-----------:|:-----------:|
| memorize-se | 30 | -5 | +40~50 | +20~30 | 85~105 |
| memorize-db | 20 | -5 | +15~20 | +15~20 | 45~55 |
| memorize-network | 25 | -4 | +20~30 | +10~15 | 51~66 |
| memorize-os | 20 | -3 | +15~20 | +5~10 | 37~47 |
| **합계** | **95** | **-17** | **+90~120** | **+50~75** | **218~273** |

## 기술 설계 방향

### 파일 구조 (신규/수정)
```
data/
  flashcards/              # 신규: 카드 데이터
    flashcards-se.json
    flashcards-db.json
    flashcards-network.json
    flashcards-os.json
features/
  flashcards/              # 신규: 카드 기능
    types.ts               # FlashCard 타입
    services/
      flashcardService.ts  # 카드 로딩/필터링
      cardConverter.ts     # 문제→카드 변환 스크립트
    hooks/
      useFlashcard.ts      # 카드 상태 관리 훅
components/
  FlashCard.tsx            # 카드 컴포넌트
  FlashCardDeck.tsx        # 카드 덱 (스와이프)
  ToggleButton.tsx         # 용어/설명 토글
app/
  quiz/
    [categoryId].tsx       # 수정: 카드/주관식 분기
    flashcard.tsx           # 신규: 카드형 암기 화면
store/
  useFlashcardStore.ts     # 신규: 카드 암기 상태
```

### 화면별 분기 로직
```
퀴즈탭 진입
  ├── 알고리즘 카테고리 (code-*, sql-*)
  │     └── 기존 주관식 문제 풀이 UI 유지
  └── 암기 카테고리 (memorize-*)
        └── 카드형 플래시카드 UI (신규)
```

## 구현 순서

### Phase 1: 데이터 변환 (우선)
1. 중복 콘텐츠 제거 스크립트 작성
2. 복합 문제 세분화 → 개별 카드 변환
3. theory-* 보기 용어 추출
4. flashcards-*.json 파일 생성

### Phase 2: 카드 UI 구현
1. FlashCard 컴포넌트 (앞뒤면 전환 애니메이션)
2. FlashCardDeck 컴포넌트 (스와이프 네비게이션)
3. 토글 버튼 (용어↔설명 반전)
4. flashcard.tsx 화면

### Phase 3: 상태 관리 & 라우팅
1. useFlashcardStore 생성 (Zustand persist)
2. 암기 상태 추적 (알아요/모르겠어요)
3. [categoryId].tsx 분기 로직 추가
4. 퀴즈탭 카테고리 목록에 진행률 표시

### Phase 4: 복습 기능
1. "모르겠어요" 카드 필터링
2. 복습 모드 진입 UI
3. 학습 통계 표시

## 범위 외 (Out of Scope)
- 스페이스드 리피티션(간격 반복) 알고리즘 (향후 확장)
- 카드 직접 편집/추가 기능
- 오디오 발음 기능
- 다른 사용자와 카드 공유
